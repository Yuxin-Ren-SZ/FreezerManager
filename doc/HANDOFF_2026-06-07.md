## What landed on `dev` (this pass)

Re-verified the `doc/CODE_REVIEW_2026-06-03.md` audit against the live tree and
found most of it already fixed. Closed the genuinely-remaining gaps in three
merges (each its own branch, TDD, verified vs live Postgres):

- **Phase 3 — `authz_version` cache invalidation (HIGH).**
  `users.authz_version` (migration 0013, SQLite + PG). Membership add/revoke/role
  change and role-permission grant/revoke bump the affected user(s) in the same
  transaction; `LocalAuthProvider` keys its permission cache on the epoch, so a
  permission downgrade takes effect on the next request instead of after the
  5-minute TTL. Disabled-user blocking was already in place.
- **Phase 6 — cross-lab integrity residuals (MED).** `container_type_id` pinned
  to the sample's lab (even when unplaced); multi-hop parent lineage cycles
  rejected (SQLite ancestry walk / PG recursive CTE). Box/item-type/project/
  checkout-event lab checks and direct self-reference were already enforced.
- **Phase 5 — repository-derived audit snapshots (MED).** `before_json`/
  `after_json` removed from `MutationContext`; repositories now derive the
  snapshot from authoritative state and pass an `AuditSnapshot` to
  `note_mutation`. A caller can no longer forge audit content. `note_mutation`
  also records a real `action` ("insert"/"update"/"soft_delete").

Already-resolved before this pass (no work needed, confirmed): Phase 2 (API-token
scope is fail-closed + lab-restricted), Phase 4 (PG audit tail is found by link
structure under an advisory lock — immune to shared `at_micros`).

Tooling: clang-tidy in CI now runs via the parallel `run-clang-tidy-17 -j$(nproc)`
(see `AGENTS.md`); only project TUs are in `compile_commands.json`.

## Next up (in priority order)

1. **Finish PG audit before-images.** SQLite captures full before/after on every
   entity; Postgres captures before+after only for Sample and Identity (the
   PHI-/auth-bearing entities). Box, Layout, ItemType, Role, Session, and
   ShareRequest record `after` + a precise `action` but not `before`. Add a
   `find_by_id` before the write in each of those `update()` methods and swap
   `detail::audit_after(entity)` → `detail::audit_change(before, entity)`. The
   forgery channel is already closed for all of them; this is audit richness, not
   a security gap. ~10 methods, mechanical.
2. **Phase 7 — run sanitizers in CI.** `build.yml` gates the Test step to the
   `dev` preset, so asan/ubsan/tsan build but never run tests. Drop the gate (or
   add a Test step per sanitizer preset) and add `--output-on-failure` to ctest.
3. **clang-tidy debt.** ~278 pre-existing errors remain (now surfaced fast by the
   parallel runner, but the step is non-blocking because `main` is unprotected).
   Mostly mechanical (`readability-identifier-length`, uppercase-literal-suffix);
   `bugprone-unchecked-optional-access` + cognitive-complexity need judgement.
4. **M2/M3 — RPC layer.** `proto/` definitions (F1) then the gRPC server (F2):
   each handler goes through `AuthMiddleware` (the RPC→permission registry already
   exists), opens a tx, works via the typed repos, and commits — the audit row is
   now produced by the repository, so handlers need not supply snapshots. API-token
   CRUD RPCs (E4) and session RPCs (D9.2) land here.
5. **Deferred auth/security follow-ups.** Password reset (E2.1) and DB-backed
   lockout (E2.2) both need `IEmailSender` (Section O) + migration 0014. PHI
   field-level encryption (H3) wraps `totp_secret_enc` and PHI custom fields;
   `Lab.is_phi_enabled` enable workflow (D10) is still unimplemented.
